## Creando mi blog en Next JS, React y Markdown

### Requerimientos previos: 

- Necesitas tener instalado [Node](https://nodejs.org/es/) 
- Y tener [NPM](https://www.npmjs.com/)


### Clona el proyecto: 

~~~
git clone https://github.com/lorenseanstewart/nextjs-mdx-blog-kit <El nombre que tu quieras>
~~~

### Inicializar el entorno de desarrollo

Para ello, lo primero, hacemos nuestro: 

~~~
npm install
~~~ 

Para que instale en la carpeta del proyecto todas las dependencias. Una vez que termine ponemos en la terminal: 

~~~
npm run dev
~~~

Con este comando se inicializa el entorno de desarrollo en: 'http://localhost:3000/' y vemos los cambios que vayamos implementando en nuestro navegador, 
cada vez que guardemos cualquiera de nuestros archivos. 

### Empieza la personalización

Verás que la arquitectura de organización de las carpetas es muy intuitiva. 
El primer paso es ir a la carpeta **config** y editar el archivo config.yml con los datos de tu proyecto. 

En el mismo definimos el autor, los metadatos, el menú, y los colores primarios para utilizar en el CSS. Muchas de las variables del CSS se definen en este 
archivo. Para mayores modificaciones, lo explico más adelante. 

### Instala CLI (Command Line Interface)

El blog, como señala en su documentación, procesa páginas en función de los metadatos encontrados y los publica extrayendo la información 
de los componentes. 

Debes asegurarte de utilizar CLI para crear todas las páginas y nuevas publicaciones.

Para instalarlo, en la terminal, dentro de la carpeta del proyecto, debemos hacer: 

~~~
npm link
~~~

Al instalar CLI podemos usar la palabra clave: **blog** para crear nuestros post. Siguiendo el siguiente formato: 

~~~
blog -t page -f miWeb // Con este comando se crea una nueva página titulada: "miWeb.js"
blog -t post -f blog-react -m // Con este comando se crea un nuevo post llamado: "blog-react"
~~~

¿Por qué uso un -m después del título del post? Porque si no identificamos el tipo de archivo, por defecto será un archivo JavaScript. 

Con el -m estoy especificando que sea markdown y ya incorpora de forma automática la cabecera con los siguientes metadatos: 

~~~

import BlogMeta from "../../components/BlogMeta";

export const meta = {
    title: "Blog en React",
    tags: ["react", "next"], 
    layout: "post",
    publishDate: "2020-04-11",
    modifiedDate: false, // "Debes usar el formato americano para las fechas: 'YYYY-MM-DD"
    seoDescription: "Aprende a crear un blog con React y Next para servir contenido estático",
    exclude: false,
    hideProgressBar: false,
}

<BlogMeta data={props} />

~~~

Y esto es lo único que necesitas para tener tu post listo. El crea automáticamente la URL usando el nombre del archivo, 
y como puedes ver, en la sección meta controlamos si queremos excluirlo (o incluirlo) en el listado del blog. 

Controlas la fecha, los tags, el title, la meta descripción. Esto con un gran asterisco ya que he tenido ciertas
complicaciones con la parte del SEO (toda una ironía). Te lo cuento más adelante. 

Debajo simplemente escribimos nuestro post. 

En la documentación oficial tenéis más ejemplos que podemos usar con el comando **blog**, pero quise reflejar lo que creo es más básico: crear un post, 
y crear una página.

### CSS

Tal como vimos al principio en el archivo 'config.yml' definimos variables generales del CSS como los colores del tema. 

Pero si queremos modificar el resto de estilos, debemos ir a la carpeta **styles** y modificar el archivo: 'index.js'. 

Yo por ejemplo, cambié la tipografía que traía por defecto para los encabezados y texto. 

Para ello, me descargué la fuente de Google Fonts y la converti al formato: .woff2 para garantizar el menor tamaño posible de mi fuente: Vollkorn y Open Sans. 
Las guardas en la carpeta: '/public/static/fonts/'. 

Después en el CSS solo debes llamarla de forma habitual con: @font-face. 

Después en el index.js del estilo, podemos modificar el CSS. Recordad que se trata de un archivo JavaScript, eso significa que todo el CSS
está encapsulado dentro de la constante: globalStyles = css.global. 

No olvidar que podemos utilizar variables del archivo de config para traer por ejemplo los colores: '${config.css.accentColor};'

### Páginas y componentes

Las diferencias entre los post y las páginas es que las páginas se almacenan en la carpeta 'pages/'. 
Los post en el directorio: 'pages/blog/'.  

Tampoco debemos poner componentes de React en el directorio de páginas, ya que Next.JS crea una ruta para cada uno de estos componentes, 
esperando que sean una página entera. Si solo tenemos un componente, lo debemos guardar en 'pages/components/'

Recordemos: Los archivos del post son markdown, y todo gracias [MDX](https://github.com/mdx-js/mdx), al igual que la página de inicio que puedes encontrar en: 'index.mdx' en la carpeta de **pages**. 

Por otra parte, las páginas son componentes (completos) de React. El template por defecto trae uno de prueba titulado: 'About' que yo eliminé de este blog. 

Utilicemos el código de este template de ejemplo, pero lo pongo en imagen usando la extensión de Visual Studio: [Polacode](https://marketplace.visualstudio.com/items?itemName=pnp.polacode)
que me parece una chulada ;-)

Lo actualizamos con las 'arrow functions', y podemos "maquetar" nuestro componente. Solo debemos agregarlo al config.yml donde definimos su ruta para el menú.

### SEO

La parte que pensé, sería más fácil del blog se me convirtió en un dolor de cabeza. En la documentación parece todo más fácil, 
pero tuve muchos problemas porque no se enviaban de forma correcta los metadatos, tal como explicaba al inicio, 
cuando se crea de forma automática 'export const meta' con todos los datos. 

El SEO en esta plantilla de inicio, se gestiona con un paquete de NPM: [Next SEO](https://www.npmjs.com/package/next-seo). Que es una maravilla. 
Simplifica mucho la gestión del SEO en los proyectos con Next.js

Ya viene en las dependencias del template, o lo puedes instalar: 

~~~
npm i next-seo
~~~

Como puedes ver en su documentación, es muy sencillo, solo debemos hacer la importación en el post o página que queramos utilizar: 

~~~
import React from 'react';
import { NextSeo } from 'next-seo';
 
export default () => (
  <>
    <NextSeo
      title="Simple Usage Example"
      description="A short description goes here."
      canonical="https://www.canonical.ie/"
    />
    <p>Simple Usage</p>
  </>
);
~~~

Y dentro del componente NextSEO definimos los metadatos y podemos agregar datos de Open Graph, podemos definir que una página no se indexe, 
podemos agregar datos estructurados, etc. 

Está genial. Según la documentación de la plantilla, los metadatos se pasan con solo modificar los datos al crear los post, 
pero no fui capaz que enviara de forma correcta. Así que tuve que usar el método directo del modulo next-seo, como se los estoy explicando. 

Si más adelante, logro resolver este punto, haré la actualización en el post.
